如何搭建博客

基于Hexo的博客搭建

一、前期准备

  1. 安装Node.js和Git

Hexo 是一个基于 Node.js 的静态网站生成器
所以首先确保你的系统中安装了 Node.js 和 Git

Node.js下载：下载

Git下载：下载

  1. 安装Hexo

使用 npm（Node.js 包管理器）在命令行中全局安装 Hexo。在命令行中输入以下命令：

npm install -g hexo-cli
  1. 初始化Hexo项目

在命令行中选择一个文件夹作为你的博客项目的根目录，然后执行以下命令：

hexo init <myblog>
cd <myblog>
  1. 安装依赖

进入你的博客项目文件夹，并安装所需的依赖：

npm install
  1. 配置Hexo

Hexo 的配置文件位于项目根目录下的 _config.yml。
你可以根据需要编辑此文件
设置博客的标题、作者、URL 等信息，以及主题、语言等其他选项

配置解析

二、编写博客

1、会使用到一些 hexo 命令，例如

  1. hexo new “name” 这将在 Hexo 项目的 source/_posts 目录中创建一个新的 Markdown 文件，用于编写你的文章。

  2. hexo generate 这将在 Hexo 项目的 public 目录中生成静态 HTML 文件。

  3. hexo server 这会启动一个本地服务器，你可以在浏览器中访问 http://localhost:4000 查看你的博客，需要注意的是在终端按下 ctrl+c 时，会停止本地服务器

  4. hexo deploy 将生成的静态网站部署到远程服务器或托管平台

  5. hexo clean 清除 Hexo 生成的临时文件和缓存文件。执行该命令后，Hexo 将会删除 public 目录下的所有文件以及其他临时文件。

  6. hexo help 查看 Hexo 的帮助信息。执行该命令后，Hexo 将会列出所有可用的命令及其说明

2、编写前的准备

在使用 Markdown 编写博客时，你可以在文章的正文之前和之后添加一些特定的内容，这些内容通常被称为 Front Matter 或 YAML Front Matter。

它们以 YAML（YAML Ain’t Markup Language）格式编写，并用于指定文章的元数据信息。

在 Hexo 中，Front Matter 通常由一对三个短划线 — 包围。

以下是一些简单的示例

---
title: 标题    # 标题
subtitle: 副标题  # 副标题
date: 2024-04-30  # 时间
top: true   # 置顶
author:      # 文章的作者或撰写者。
comments:true     # 是否允许评论
image:        # 图片资源路径
categories:   # 分类
  - 技术
tags:     # 标签
  - Hexo
  - Markdown
---

三、部署博客

当你满意生成的网站后，可以将其部署到你选择的托管平台上。

Hexo 支持多种部署方式，包括 Gitee Pages、 GitHub Pages、Netlify、Vercel 等。
具体的部署方法取决于你选择的托管平台，你可以查阅相关文档以了解详细步骤。
